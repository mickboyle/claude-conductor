# Markdown Style Guide

## File Structure

```markdown
# Title (H1)

Brief introduction paragraph.

## Section (H2)

Content...

### Subsection (H3)

More specific content...
```

## Headings

- Use ATX-style headings (`#` not underlines)
- Only one H1 per file (the title)
- Don't skip levels (H2 â†’ H4)
- Use sentence case: "Quick start guide" not "Quick Start Guide"

## Lists

**Unordered lists:** Use `-` consistently

```markdown
- Item one
- Item two
  - Nested item
```

**Ordered lists:** Use `1.` for all items (auto-numbering)

```markdown
1. First step
1. Second step
1. Third step
```

## Code

**Inline code:** Use backticks for commands, file names, values

```markdown
Run `git status` to check.
```

**Code blocks:** Use fenced blocks with language identifier

````markdown
```bash
git add .
git commit -m "Message"
```
````

## Tables

```markdown
| Header 1 | Header 2 |
|----------|----------|
| Cell 1   | Cell 2   |
```

- Align columns for readability in source
- Use `---` for separator (min 3 dashes)

## Links

**Internal links:** Use relative paths

```markdown
See [Product Definition](./product.md)
```

**External links:** Include full URL

```markdown
[Claude Code](https://claude.com/claude-code)
```

## Emphasis

- *Italics* for terms or light emphasis
- **Bold** for important points
- `Code` for technical terms
- Don't combine (no ***bold italic***)

## Templates

Use double curly braces for placeholders:

```markdown
**Track ID:** {{TRACK_ID}}
**Created:** {{DATE}}
```

## Checkboxes

For task lists:

```markdown
- [ ] Pending task
- [~] In progress
- [x] Complete task
- [!] Blocked
```

## Document Footer

End documents with metadata:

```markdown
---

*Generated by Claude Conductor on {{DATE}}*
```
